🏗️ Project Architecture Overview

MCP Guardian is a security governance proxy for Model Context Protocol (MCP) infrastructure. It sits between AI clients (Claude Desktop, Cline, Cursor) and MCP servers, enforcing active security policies, tracking real token costs, and monitoring health.

⚠️ Remaining Gaps & Known Limitations

P1 Proxy Backpressure

When MCP servers produce responses faster than AI clients consume, Node.js buffers grow unbounded. Needs highWaterMark on stdin/stdout pipes and drain event handling.

P1 Scanner Concurrency Limit

20+ servers spawn 100+ simultaneous HTTP requests to OSV.dev/NVD. Needs a p-limit concurrency cap.

P1 Default Policy Regex Anchoring

rm\s+-rf matches "alarm_rf" substring. Patterns need \b word boundaries.

P2 MSW Test Mocks

Tests calling OSV.dev/NVD fail in air-gapped environments. Need Mock Service Worker for offline CI.

P2 Coverage Gates

No minimum line/branch coverage enforced in CI. Need vitest coverage thresholds.

P2 PyPI/pip CVE Scanning

Only npm purl format supported. Python MCP servers (uvx-based) get no CVE data.

P2 MCP Protocol Version Negotiation

Health probe sends fixed initialize request. Newer MCP spec versions (2025-11) may fail.

P2 Helm Chart Not Published

helm repo add URL still marked "(future)" in README.

P2 Dashboard TLS, Audit Immutability, Certificate Rotation, Live Typo-Squat List, Branded Types

Enterprise features deferred to P2 roadmap.
